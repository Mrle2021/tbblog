# WeChat Draft Sync

This document records the current workflow for creating WeChat Official Account drafts from Hexo blog posts.

The sync creates or updates a WeChat draft. It does not publish articles automatically. Final review and publishing still happen in the WeChat backend.

## Source Of Truth

- Blog Markdown posts live in `source/_posts/`.
- Local images live under `source/images/`.
- Sync entrypoint: `tools/wechat-draft-sync.js`.
- Main command: `npm run wechat:draft`.
- Draft state file: `.wechat-drafts.json`.

`.wechat-drafts.json` is local operational state and is ignored by git. It stores the WeChat draft media ID for each post slug so future runs can update the same draft instead of creating a new one.

## Environment Variables

Create `.env` locally or on the VPS checkout with:

```bash
WECHAT_APP_ID=your_app_id
WECHAT_APP_SECRET=your_app_secret
WECHAT_AUTHOR=Wanger
```

Required:

- `WECHAT_APP_ID`
- `WECHAT_APP_SECRET`

Optional:

- `WECHAT_AUTHOR`: author name shown in the WeChat draft. If omitted, the script sends an empty author.

Do not commit `.env`.

## Local Dry Run

Use dry-run before calling the WeChat API:

```bash
POST=source/_posts/ai-reading-world-weekly-news-workflow.md npm run wechat:draft -- --dry-run
```

Dry-run behavior:

- parses the post frontmatter and Markdown body;
- converts Markdown to WeChat-compatible HTML;
- removes unsupported links and code tags;
- keeps image references local;
- prints the draft payload as JSON;
- does not upload images;
- does not call the WeChat draft API;
- does not write `.wechat-drafts.json`.

Useful safety check:

```bash
POST=source/_posts/ai-reading-world-weekly-news-workflow.md npm run wechat:draft -- --dry-run > /tmp/wechat-draft-dry-run.json
node -e "const fs=require('fs'); const raw=fs.readFileSync('/tmp/wechat-draft-dry-run.json','utf8'); const json=JSON.parse(raw.slice(raw.indexOf('{'))); const c=json.article.content; console.log(JSON.stringify({links:(c.match(/<a\\b/g)||[]).length, imgs:(c.match(/<img\\b/g)||[]).length, pre:(c.match(/<pre\\b/g)||[]).length, code:(c.match(/<code\\b/g)||[]).length, oldRootLinks:(c.match(/\\/tbblog\\//g)||[]).length, prompt:(c.match(/data-wechat-block=\"prompt\"/g)||[]).length, related:(c.match(/data-wechat-block=\"related\"/g)||[]).length}, null, 2));"
```

Expected for the current sample post:

```json
{
  "links": 0,
  "imgs": 5,
  "pre": 0,
  "code": 0,
  "oldRootLinks": 0,
  "prompt": 1,
  "related": 1
}
```

## VPS Real Sync

The VPS does not need host Node.js for this workflow. Run the command inside the Docker Compose `builder` service:

```bash
ssh root@192.210.213.108 'cd /opt/tbblog && docker compose run --rm builder sh -lc '\''POST=source/_posts/ai-reading-world-weekly-news-workflow.md npm run wechat:draft'\'''
```

Expected successful output includes:

```json
{
  "dryRun": false,
  "draftMediaId": "..."
}
```

After success, open the WeChat backend and review the generated draft manually.

## Current Typography Rules

WeChat draft HTML is generated by `tools/wechat/html-converter.js`.

Current rules:

- `h2`: chapter heading with a green left accent line.
- `p`: dark-gray body text, 15px font size, larger line height.
- `blockquote`: light green callout with a left border.
- `ul` and `li`: adjusted indentation and vertical spacing.
- `img`: full-width responsive image, vertical spacing, small rounded corners.
- Links: unwrapped, keeping only visible link text.
- Code blocks: converted to ordinary paragraphs with `<br>` line breaks.
- `class` attributes: removed.
- "我的固定提示词": following content is wrapped in a light blue card.
- "相关阅读": following list is wrapped in a light warm recommendation block.

The generated article intentionally avoids:

- `<a>`
- `<pre>`
- `<code>`
- site-relative link URLs such as `/tbblog/...`

These restrictions reduce the chance of WeChat draft API content rejection.

## Common Errors

### IP Not In WeChat Whitelist

Symptom:

- WeChat API returns an invalid credential or IP whitelist related error.

Fix:

- Add the VPS public IP `192.210.213.108` to the WeChat Official Account API IP whitelist.
- Run the real sync from the VPS container, not from a random local network, unless that local IP is also whitelisted.

### WebP Cover Or Inline Images

Previous symptom:

```text
uploadThumb failed: errcode=40113 errmsg=unsupported file type
```

Cause:

- The blog uses WebP images, but WeChat thumb upload does not accept WebP.

Current behavior:

- `tools/wechat/image-prep.js` converts thumbnails to small JPG files before upload.
- Unsupported inline images are converted to JPG before upload.

If this fails again, run:

```bash
npm run test:wechat
```

and inspect `tools/wechat/image-prep.js`.

### 45166 Invalid Content

Previous symptom:

```text
addDraft failed: errcode=45166 errmsg=invalid content
```

Cause:

- WeChat rejected HTML content that included unsupported or risky structures, especially raw links, site-relative URLs, and code block markup.

Current behavior:

- Links are unwrapped and only their visible text remains.
- `<pre>` and `<code>` are removed.
- `/tbblog/...` link remnants should be zero in dry-run inspection.

Run the safety check in the Local Dry Run section. If any of `links`, `pre`, `code`, or `oldRootLinks` is not `0`, fix `tools/wechat/html-converter.js` before doing a live sync.

### Draft Not Updating As Expected

The script updates an existing draft when `.wechat-drafts.json` has a `draftMediaId` for the post slug.

If the WeChat backend no longer has that draft, the update may fail. In that case:

1. Inspect `.wechat-drafts.json` on the machine running the sync.
2. Remove the stale entry for the post slug.
3. Run the real sync again to create a fresh draft.

Do not commit `.wechat-drafts.json`.
